# Contributing

Thank you for considering contributing to this project!
Contributions of all kinds are welcome, including code, documentation, design, and ideas.

## License

This project is licensed under the **[MIT license](LICENSE)**.

By submitting a contribution (pull request, commit, or code included in an issue), you agree that:

- Your contribution will be licensed under the **[MIT license](LICENSE)**
- You have the right to submit the contribution (it is your original work or you have appropriate permission)

If you do not agree with these terms, we unfortunately cannot accept the contribution.

## What you can contribute

You can contribute with:

- Bug fixes
- New sections, elements, components, or improvements to existing ones
- Performance improvements
- Refactors
- UI/UX improvements
- Documentation
- Accessibility improvements

## How to contribute

1. Fork the repository
2. Create a branch from `main`

```bash
git switch -c feature/feature-name
```

3. Make clear and focused commits
4. Open a Pull Request explaining **what was done** and **why**

## Best practices

- Keep consistency with the existing code style
- Avoid overly large commits
- Prefer readable code over clever code
- Comments are welcome when they help explain decisions
